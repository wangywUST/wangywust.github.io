## Notation and Variable Definitions

This chapter uses the notation of the previous three lectures. The new emphasis is that a random signal is represented by a **linear model** driven by an excitation process, usually white noise.

| $n$ | Discrete-time sample index |
| $N$ | Number of observed samples, usually $x(0),x(1),\ldots,x(N-1)$ |
| $P$ | Number of poles / AR order / denominator order |
| $Q$ | Number of zeros / MA order / numerator order |
| $p$ | Generic model order, often used when discussing AR($p$) |
| $q$ | Generic MA order |

# §0 Modeling Viewpoint and Chapter Roadmap

> 📖 Textbook §4.1 (Introduction); §9.1 (The Modeling Process: Theory and Practice)

## 0.1 What Is a Signal Model?

A **signal model** is a simplified mathematical description of how a signal could have been generated.

In Chapters 1 and 2, we described signals by their time-domain samples, transforms, correlations, and spectra. In Chapter 3, we used linear prediction and lattice structures to estimate or whiten predictable components. In this chapter, we turn those ideas into a modeling framework:

> Instead of only asking *what is the spectrum of this random signal?*, we ask:  
> **Can this signal be generated by passing white noise through a simple linear system?**

The central modeling picture is

$$w(n) \longrightarrow H(z) \longrightarrow x(n),$$

where

- $w(n)$ is white noise, representing new unpredictable information,
- $H(z)$ is a stable linear time-invariant system,
- $x(n)$ is the observed random signal.

If $H(z)$ is known, then the second-order behavior of $x(n)$ is known. If $H(z)$ is unknown, then signal modeling means estimating $H(z)$ from data.

## 0.2 Why Model Random Signals This Way?

Linear signal models are useful because they replace a complicated random signal by a small number of parameters.

For example, a long data record might contain $N=1000$ samples, but an AR(4) model uses only four AR coefficients and one variance parameter. This is a strong simplification. It is useful only if the model captures the important structure of the data.

Parametric modeling is especially attractive when:

| Situation | Why a Parametric Model Helps |
|----------|------------------------------|
| Short data record | A model can extrapolate correlation beyond observed lags |
| Narrow spectral peaks | Poles can represent resonances sharply |
| Spectral valleys or notches | Zeros can represent deep nulls |
| Compression | Transmit model parameters instead of all samples |
| Prediction | Use the model to forecast future samples |
| Whitening | Remove predictable correlation and analyze residuals |

The price is that the model may be wrong. A wrong model can create artificial spectral peaks, miss true notches, or give misleading interpretations.

## 0.3 Deterministic Modeling versus Random Signal Modeling

It is helpful to distinguish two related but different problems.

### Deterministic Signal Modeling

For deterministic data, we often choose model coefficients to minimize a finite-data error energy:

$$\sum_n |e(n)|^2.$$

This is a least-squares problem. The data sequence is treated as fixed.

### Random Signal Modeling

For a random process, the ideal criterion is usually a mean-square quantity such as

$$E\{|e(n)|^2\}.$$

This expectation depends on the underlying probability law. In practice, we do not know the probability law, so we estimate correlations or use finite-data least-squares approximations.

The two viewpoints meet in practical algorithms:

| Random-Process Ideal | Finite-Data Implementation |
|---------------------|----------------------------|
| Use $r_x(l)=E\{x(n)x^\ast(n-l)\}$ | Estimate $\hat r_x(l)$ from data |
| Minimize $E\{\lvert e(n)\rvert^2\}$ | Minimize $\sum_n \lvert e(n)\rvert^2$ |
| Obtain theoretical AR/MA/ARMA coefficients | Obtain estimated coefficients $\hat a_k,\hat d_k$ |

## 0.4 The Three-Stage Modeling Process

The textbook organizes practical signal modeling as an iterative process:

1. choose a model structure and order,
2. estimate model parameters,
3. validate the model.

> ![Figure 0.1](./CourseADSP2026/Fig/Chapter_4/fig_5_1_textbook_fig_9_1_p447.png)
>
> *Figure 0.1 (Textbook Fig. 9.1, p. 447): Steps in the signal model building process. Modeling is not a one-pass formula. It is an iterative loop: choose a model, estimate parameters, validate residuals/performance, and revise if needed.*

This loop is important pedagogically. Many students think model order selection is just a technical detail. It is not. The order determines the complexity and behavior of the model.

A good model should satisfy three requirements:

| Requirement | Meaning |
|------------|---------|
| Fits the observed data | The model reproduces key correlations and spectral features |
| Has interpretable residuals | The residual should look approximately white if the model has captured the predictable structure |
| Serves the application | A model for spectrum estimation, coding, prediction, or detection may require different tradeoffs |

## 0.5 Chapter Roadmap

This chapter proceeds in the following logical order.

First, we define the broad class of white-noise-driven linear models. Second, we restrict the system $H(z)$ to rational forms, producing AP, AZ, and PZ models. Third, we identify the signatures of these models in the autocorrelation, partial autocorrelation, and spectrum. Fourth, we discuss practical algorithms for estimating AR, MA, and ARMA parameters. Finally, we explain how these models are used for power spectrum estimation.

# §1 Linear Random Signal Models

> 📖 Textbook §4.1.1 (Linear Nonparametric Signal Models); §4.1.2 (Parametric Pole-Zero Signal Models); §2.4.4 (Spectral Factorization)

## 1.1 White Noise Through an LTI System

The most basic linear random signal model is

$$x(n)=\sum_{k=-\infty}^{\infty}h(k)w(n-k),$$

where $w(n)$ is zero-mean white noise with

$$r_w(l)=\sigma_w^2\delta(l),\qquad R_w(e^{j\omega})=\sigma_w^2.$$

Because the input spectrum is flat, the shape of the output spectrum is entirely controlled by the filter:

$$\boxed{R_x(e^{j\omega})=\sigma_w^2|H(e^{j\omega})|^2}.$$

Similarly, the output autocorrelation is

$$\boxed{r_x(l)=\sigma_w^2\sum_{k=-\infty}^{\infty}h(k)h^\ast(k-l)}.$$

Equivalently, in the z-domain,

$$\boxed{R_x(z)=\sigma_w^2H(z)H^\ast\left(\frac{1}{z^\ast}\right)}.$$

The interpretation is simple:

- $w(n)$ supplies uncorrelated innovations,
- $H(z)$ introduces memory and spectral coloring,
- $x(n)$ becomes correlated even though $w(n)$ is not.

> ![Figure 1.1](./CourseADSP2026/Fig/Chapter_4/fig_0_1_textbook_fig_4_1_p150.png)
>
> *Figure 1.1 (Textbook Fig. 4.1, p. 150): Signal models with continuous and discrete power spectrum densities. White noise passed through an LTI system produces a continuous shaped PSD; a white harmonic process passed through a filter produces shaped spectral lines.*

## 1.2 Nonparametric and Parametric Models

There are two ways to specify the filter $H(z)$.

### Nonparametric Model

If $H(z)$ is described by its impulse response $h(n)$ without restricting the number of parameters, the model is nonparametric.

For example,

$$x(n)=\sum_{k=-\infty}^{\infty}h(k)w(n-k)$$

may require infinitely many values $h(k)$.

This is very flexible, but it is not compact.

### Parametric Model

If $H(z)$ is described by a finite-order rational function,

$$H(z)=\frac{D(z)}{A(z)},$$

then only a finite set of coefficients is needed. This is the central topic of this chapter.

The finite parameter set usually looks like

$$\{a_1,\ldots,a_P,d_1,\ldots,d_Q,\sigma_w^2\}.$$

Parametric models are less flexible than arbitrary impulse responses, but they are easier to estimate, interpret, and use.

## 1.3 Coloring and Whitening Filters

If $x(n)$ is generated by

$$x(n)=H(z)w(n),$$

then $H(z)$ is called the **synthesis filter** or **coloring filter**. It turns white noise into colored noise.

If the inverse system exists and is stable,

$$H_I(z)=\frac{1}{H(z)},$$

then

$$w(n)=H_I(z)x(n).$$

This inverse filter is called the **analysis filter** or **whitening filter**.

> ![Figure 1.2](./CourseADSP2026/Fig/Chapter_4/fig_1_1_textbook_fig_4_2_p152.png)
>
> *Figure 1.2 (Textbook Fig. 4.2, p. 152): Synthesis/coloring and analysis/whitening filters used in innovations representation. The forward system generates the correlated process; the inverse system recovers the innovation sequence if the model is minimum phase.*

This figure is one of the most important conceptual figures in the chapter. It explains why modeling and prediction are connected:

- modeling: find $H(z)$ so that $x(n)=H(z)w(n)$,
- whitening: apply $1/H(z)$ so that the residual is white,
- prediction: if the residual is the innovation, then everything predictable has been removed.

## 1.4 Minimum Phase and Innovations

The innovations interpretation requires more than stability. We usually want $H(z)$ to be **minimum phase**, meaning that both $H(z)$ and its inverse are causal and stable.

If $H(z)$ is minimum phase, then the current innovation $w(n)$ is the genuinely new information in $x(n)$ that cannot be predicted from past samples.

For a minimum-phase model,

$$x(n+1)=\text{linear combination of past }x(n),x(n-1),\ldots+w(n+1).$$

This equation has a direct meaning:

| Term | Meaning |
|------|---------|
| Past-dependent part | Information already contained in earlier samples |
| $w(n+1)$ | New information arriving at time $n+1$ |

This is why all-pole modeling and linear prediction are so closely related. If the prediction error is white, it is an estimate of the innovations sequence.

## 1.5 Spectral Factorization

Suppose a WSS process has a continuous PSD $R_x(e^{j\omega})$. If it can be written as

$$R_x(e^{j\omega})=\sigma_w^2|H_{\min}(e^{j\omega})|^2,$$

where $H_{\min}(z)$ is stable, causal, and minimum phase, then $H_{\min}(z)$ is a spectral factor of $R_x$.

In z-domain notation,

$$\boxed{R_x(z)=\sigma_w^2H_{\min}(z)H_{\min}^\ast\left(\frac{1}{z^\ast}\right)}.$$

The idea is this:

> A power spectrum is magnitude-squared information.  
> Spectral factorization recovers a causal stable filter whose magnitude-squared response equals that spectrum.

This is not always trivial, because a magnitude response does not uniquely determine phase. The minimum-phase factor provides a canonical causal and invertible choice.

## 1.6 Spectral Flatness Measure

The textbook defines the spectral flatness measure as

$$\mathrm{SFM}_x=\frac{\exp\left\{\frac{1}{2\pi}\int_{-\pi}^{\pi}\ln R_x(e^{j\omega})d\omega\right\}}{\frac{1}{2\pi}\int_{-\pi}^{\pi}R_x(e^{j\omega})d\omega}.$$

For a regular process generated by a minimum-phase filter, this can be interpreted as

$$\boxed{\mathrm{SFM}_x=\frac{\sigma_w^2}{\sigma_x^2}}.$$

Thus:

- $\mathrm{SFM}_x=1$ means the spectrum is perfectly flat, i.e., white noise.
- $\mathrm{SFM}_x\ll 1$ means the spectrum is strongly colored.

This quantity is useful in modeling because whitening should increase spectral flatness. If the residual of a model has SFM close to one, the model has removed much of the spectral structure.

# §2 Parametric Pole-Zero Models

> 📖 Textbook §4.1.2 (Parametric Pole-Zero Signal Models); §4.2 (All-Pole Models); §4.3 (All-Zero Models); §4.4 (Pole-Zero Models)

## 2.1 The General PZ($P,Q$) Model

A rational parametric model is described by the difference equation

$$\boxed{x(n)+\sum_{k=1}^{P}a_kx(n-k)=\sum_{k=0}^{Q}d_kw(n-k)}.$$

Taking the z-transform gives

$$\boxed{H(z)=\frac{X(z)}{W(z)}=\frac{D(z)}{A(z)}=\frac{\sum_{k=0}^{Q}d_kz^{-k}}{1+\sum_{k=1}^{P}a_kz^{-k}}}.$$

Equivalently, in terms of poles and zeros,

$$H(z)=d_0\frac{\prod_{k=1}^{Q}(1-z_kz^{-1})}{\prod_{k=1}^{P}(1-p_kz^{-1})}.$$

> ![Figure 2.1](./CourseADSP2026/Fig/Chapter_4/fig_2_1_textbook_fig_4_3_p155.png)
>
> *Figure 2.1 (Textbook Fig. 4.3, p. 155): Block diagram representation of a parametric rational signal model. The output process is fully specified, in the second-order sense, by the model coefficients and the excitation variance.*

The model becomes stationary when the synthesis filter is stable. For a causal rational model, this requires all poles to lie inside the unit circle:

$$|p_k|<1,\qquad k=1,\ldots,P.$$

If we also want the inverse whitening filter to be stable, all zeros should lie inside the unit circle as well.

## 2.2 Three Important Special Cases

The three major model families are obtained by choosing $P$ and $Q$.

### All-Pole / AR Model

If $Q=0$, then

$$H(z)=\frac{d_0}{A(z)}.$$

The time-domain equation is

$$\boxed{x(n)=-\sum_{k=1}^{P}a_kx(n-k)+d_0w(n)}.$$

This is called an AP($P$) model in system language and an AR($P$) model in random-process language.

### All-Zero / MA Model

If $P=0$, then

$$H(z)=D(z).$$

The time-domain equation is

$$\boxed{x(n)=\sum_{k=0}^{Q}d_kw(n-k)}.$$

This is called an AZ($Q$) model in system language and an MA($Q$) model in random-process language.

### Pole-Zero / ARMA Model

If $P>0$ and $Q>0$, then

$$H(z)=\frac{D(z)}{A(z)}.$$

The time-domain equation is

$$\boxed{x(n)=-\sum_{k=1}^{P}a_kx(n-k)+\sum_{k=0}^{Q}d_kw(n-k)}.$$

This is called a PZ($P,Q$) model in system language and an ARMA($P,Q$) model in random-process language.

## 2.3 Why Poles and Zeros Matter

The pole-zero form tells us what kind of spectral shape the model can produce.

### Poles Create Peaks

A pole near the unit circle produces a resonance. If the pole is at

$$p=re^{j\theta},\qquad r<1,$$

then the spectrum tends to have a peak near frequency $\theta$. As $r$ approaches 1, the peak becomes sharper and the impulse response decays more slowly.

This is why all-pole models are good at representing narrow spectral peaks.

### Zeros Create Valleys or Notches

A zero near the unit circle suppresses energy near its angle. If a zero is exactly on the unit circle, it can produce a spectral null.

This is why all-zero models are good at representing spectral valleys and notches.

### Pole-Zero Models Combine Both

An ARMA model can use poles for peaks and zeros for notches. This often gives a more accurate spectrum with fewer parameters than a pure AR or pure MA model.

The tradeoff is that ARMA estimation is more difficult, because the unknown excitation appears inside a recursive relationship.

## 2.4 Short-Memory Behavior

For a causal pole-zero model with distinct poles, the impulse response can be written as a combination of impulses, real exponentials, and damped sinusoids:

$$h(n)=\sum_jB_j\delta(n-j)+\sum_kA_kp_k^nu(n)+\sum_iC_ir_i^n\cos(\omega_in+\phi_i)u(n).$$

This expression explains the short-memory behavior:

- all-zero models have finite impulse responses,
- stable all-pole and pole-zero models have exponentially decaying impulse responses,
- poles closer to the unit circle produce slower decay and longer memory.

Therefore, stable finite-order pole-zero models are usually appropriate for stationary signals with **short memory**.

## 2.5 A Detailed Example: AP(2)

The second-order all-pole model is important because it already shows the main geometric ideas.

Let

$$H(z)=\frac{d_0}{1+a_1z^{-1}+a_2z^{-2}}=\frac{d_0}{(1-p_1z^{-1})(1-p_2z^{-1})}.$$

The model is stable and minimum phase when both poles are inside the unit circle. In the $(a_1,a_2)$ parameter plane, this gives a triangular admissible region.

> ![Figure 2.2](./CourseADSP2026/Fig/Chapter_4/fig_2_2_textbook_fig_4_6_p168.png)
>
> *Figure 2.2 (Textbook Fig. 4.6, p. 168): Minimum-phase region for the AP(2) model in the $(a_1,a_2)$ parameter space. The triangular region corresponds to stable second-order all-pole filters; the upper curved boundary separates real and complex-conjugate pole cases.*

If the poles are complex conjugates,

$$p_{1,2}=re^{\pm j\theta},$$

then

$$a_1=-2r\cos\theta,\qquad a_2=r^2.$$

The impulse response is a damped sinusoid. The autocorrelation is a damped cosine. The PSD has a resonant peak.

> ![Figure 2.3](./CourseADSP2026/Fig/Chapter_4/fig_2_3_textbook_fig_4_7_p170.png)
>
> *Figure 2.3 (Textbook Fig. 4.7, p. 170): Sample realization, impulse response, autocorrelation, and spectrum of an AP(2) model with complex conjugate poles. This figure connects the pole picture with time-domain memory and spectral resonance.*

This figure is useful for teaching because it links four descriptions of the same model:

| Description | What It Shows |
|------------|---------------|
| Sample realization | The output looks pseudoperiodic |
| Impulse response | A damped oscillation |
| Autocorrelation | A damped cosine |
| Spectrum | A peak near the pole angle |

# §3 Model Signatures: ACS, PACS, and Spectrum

> 📖 Textbook §4.2–§4.4; Table 4.1

## 3.1 Why Model Signatures Matter

Before estimating parameters, we should ask what model family is appropriate.

The textbook emphasizes three diagnostic objects:

1. autocorrelation sequence (ACS),
2. partial autocorrelation sequence (PACS),
3. power spectrum.

The main idea is:

> Different model families leave different fingerprints in ACS, PACS, and PSD.

These fingerprints guide model selection.

## 3.2 All-Pole / AR Signature

For an AR($P$) process,

$$x(n)+\sum_{k=1}^{P}a_kx(n-k)=w(n).$$

The autocorrelation satisfies a homogeneous difference equation after lag zero:

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=0,\qquad l\ge 1,$$

with appropriate equations at small lags.

Therefore, the ACS generally tails off as damped exponentials or damped sinusoids.

The PACS, however, cuts off after lag $P$:

$$k_m=0,\qquad m>P.$$

This gives the classic AR diagnostic:

| Diagnostic | AR($P$) Behavior |
|-----------|------------------|
| ACS | Tails off |
| PACS | Cuts off after lag $P$ |
| Spectrum | Good at matching peaks |

## 3.3 All-Zero / MA Signature

For an MA($Q$) process,

$$x(n)=\sum_{k=0}^{Q}d_kw(n-k).$$

Because $x(n)$ depends only on a finite window of white-noise samples, the autocorrelation has finite duration:

$$r_x(l)=0,\qquad |l|>Q.$$

However, the PACS usually tails off.

Thus the MA diagnostic is dual to the AR diagnostic:

| Diagnostic | MA($Q$) Behavior |
|-----------|------------------|
| ACS | Cuts off after lag $Q$ |
| PACS | Tails off |
| Spectrum | Good at matching valleys/notches |

A first-order all-zero model can look low-pass or high-pass depending on the sign of its coefficient.

> ![Figure 3.1](./CourseADSP2026/Fig/Chapter_4/fig_3_1_textbook_fig_4_9_p175.png)
>
> *Figure 3.1 (Textbook Fig. 4.9, p. 175): Output realization, ACS, PACS, and spectrum of an AZ(1) model with $d_1=0.95$. The ACS cuts off after lag 1, while the PACS decays gradually.*

> ![Figure 3.2](./CourseADSP2026/Fig/Chapter_4/fig_3_2_textbook_fig_4_10_p175.png)
>
> *Figure 3.2 (Textbook Fig. 4.10, p. 175): Output realization, ACS, PACS, and spectrum of an AZ(1) model with $d_1=-0.95$. Changing the sign of the zero coefficient changes the spectral shape from low-pass-like to high-pass-like.*

## 3.4 Pole-Zero / ARMA Signature

For an ARMA($P,Q$) process,

$$x(n)+\sum_{k=1}^{P}a_kx(n-k)=w(n)+\sum_{k=1}^{Q}d_kw(n-k).$$

The AR part produces a tailing autocorrelation, while the MA part affects the short lags and can create spectral valleys.

Therefore, both ACS and PACS usually tail off:

| Diagnostic | ARMA($P,Q$) Behavior |
|-----------|-----------------------|
| ACS | Tails off after the MA-dominated short-lag region |
| PACS | Tails off after the AR-dominated short-order region |
| Spectrum | Can match both peaks and valleys |

The PZ(1,1) model already shows that the admissible parameter region can be more complicated than the AR or MA cases.

> ![Figure 3.3](./CourseADSP2026/Fig/Chapter_4/fig_4_1_textbook_fig_4_12_p180.png)
>
> *Figure 3.3 (Textbook Fig. 4.12, p. 180): Minimum-phase and positive-definiteness regions for the PZ(1,1) model. The coefficient-domain condition and the autocorrelation-domain condition are not the same geometric object.*

## 3.5 Summary of Model Family Signatures

The textbook's Table 4.1 can be summarized as follows.

| Property | AP($P$) / AR($P$) | AZ($Q$) / MA($Q$) | PZ($P,Q$) / ARMA($P,Q$) |
|----------|-------------------|-------------------|--------------------------|
| System function | $1/A(z)$ | $D(z)$ | $D(z)/A(z)$ |
| Recursive representation | Finite | Infinite if invertible | Infinite |
| Nonrecursive representation | Infinite | Finite | Infinite |
| Stability | Poles inside unit circle | Always stable as FIR | Poles inside unit circle |
| Invertibility | Always, for no zeros | Zeros inside unit circle | Zeros inside unit circle |
| ACS | Infinite duration; tails off | Finite duration; cuts off | Infinite duration; tails off |
| PACS | Finite duration; cuts off | Infinite duration; tails off | Infinite duration; tails off |
| Spectral strength | Peaks | Notches/valleys | Peaks and valleys |

The two most useful practical rules are:

1. If PACS cuts off but ACS tails off, try an AR model.
2. If ACS cuts off but PACS tails off, try an MA model.

If both tail off and the spectrum has both resonant peaks and clear notches, try an ARMA model.

# §4 ARMA Modeling and Modified Yule-Walker Ideas

> 📖 Textbook §4.4 (Pole-Zero Models); §9.3 (Estimation of Pole-Zero Models)

## 4.1 The ARMA Modeling Problem

The ARMA($P,Q$) model is

$$\boxed{x(n)+\sum_{k=1}^{P}a_kx(n-k)=w(n)+\sum_{k=1}^{Q}d_kw(n-k)}.$$

Equivalently,

$$A(z)x(n)=D(z)w(n).$$

The PSD is

$$\boxed{R_x(e^{j\omega})=\sigma_w^2\frac{|D(e^{j\omega})|^2}{|A(e^{j\omega})|^2}}.$$

The goal is to estimate

$$\{a_1,\ldots,a_P,d_1,\ldots,d_Q,\sigma_w^2\}$$

from a finite observation $x(0),x(1),\ldots,x(N-1)$.

This is harder than AR modeling because both the numerator and denominator are unknown. The denominator creates recursion in $x(n)$, while the numerator contains the unobserved excitation $w(n)$.

## 4.2 A Useful Intermediate Process

Define

$$c(n)=A(z)x(n)=x(n)+\sum_{k=1}^{P}a_kx(n-k).$$

For a true ARMA($P,Q$) process,

$$c(n)=D(z)w(n)=w(n)+\sum_{k=1}^{Q}d_kw(n-k).$$

Thus $c(n)$ is an MA($Q$) process.

This observation is the basis of modified Yule-Walker methods:

1. Use long-lag autocorrelation equations to estimate the AR part $A(z)$.
2. Filter $x(n)$ by the estimated $A(z)$ to obtain an estimated MA process $\hat c(n)$.
3. Estimate the MA part $D(z)$ from the autocorrelation or spectrum of $\hat c(n)$.

## 4.3 Deriving the Modified Yule-Walker Equations

Start with

$$x(n)+\sum_{k=1}^{P}a_kx(n-k)=c(n),$$

where $c(n)$ is MA($Q$).

Multiply both sides by $x^\ast(n-l)$ and take expectation:

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=r_{cx}(l).$$

For lags larger than $Q$, the current MA excitation $c(n)$ is uncorrelated with $x(n-l)$, so

$$r_{cx}(l)=0,\qquad l>Q.$$

Therefore,

$$\boxed{r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=0,\qquad l>Q.}$$

This is the modified Yule-Walker equation for ARMA modeling.

Using $l=Q+1,Q+2,\ldots,Q+P$, we obtain $P$ linear equations for the $P$ AR coefficients:

$$\begin{bmatrix}
r_x(Q) & r_x(Q-1) & \cdots & r_x(Q-P+1)\\
r_x(Q+1) & r_x(Q) & \cdots & r_x(Q-P+2)\\
\vdots & \vdots & \ddots & \vdots\\
r_x(Q+P-1)&r_x(Q+P-2)&\cdots&r_x(Q)
\end{bmatrix}
\begin{bmatrix}a_1\\a_2\\\vdots\\a_P\end{bmatrix}
=-\begin{bmatrix}r_x(Q+1)\\r_x(Q+2)\\\vdots\\r_x(Q+P)\end{bmatrix}.$$

This system is Toeplitz-like and can be solved efficiently by specialized algorithms when the structure is exactly Toeplitz.

## 4.4 Least-Squares MYWE

The basic MYWE method uses exactly $P$ equations. In noisy finite-data settings, this can be fragile.

A more robust approach is to use more lags than necessary:

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)\approx 0,\qquad l=Q+1,\ldots,Q+M,$$

where $M>P$.

This gives an overdetermined system

$$\mathbf{R}\mathbf{a}\approx-\mathbf{r}.$$

The least-squares solution is

$$\boxed{\hat{\mathbf{a}}=-(\mathbf{R}^H\mathbf{R})^{-1}\mathbf{R}^H\mathbf{r}}.$$

This is the LS-MYWE method. It is usually more stable under correlation-estimation error because it averages information across many lags.

## 4.5 Estimating the MA Part After the AR Part

Once $\hat A(z)$ is known, form

$$\hat c(n)=\hat A(z)x(n).$$

Then estimate the autocorrelation $\hat r_c(l)$ of $\hat c(n)$. Since $c(n)$ should be MA($Q$), its autocorrelation should be zero for $|l|>Q$.

The relation

$$R_c(z)=\sigma_w^2D(z)D^\ast\left(\frac{1}{z^\ast}\right)$$

means that $D(z)$ can be recovered by spectral factorization of $R_c(z)$.

Practical implementations usually choose the minimum-phase factor so that the resulting MA model is invertible.

## 4.6 Textbook Least-Squares View of PZ Estimation

The textbook emphasizes least-squares pole-zero estimation. If the excitation $w(n)$ were known, then PZ estimation would be linear.

Define

$$\mathbf{z}(n)=[-x(n),\ldots,-x(n-P+1),w(n),\ldots,w(n-Q+1)]^T$$

and

$$\mathbf{c}=[a_1,\ldots,a_P,d_1,\ldots,d_Q]^T.$$

Then

$$x(n)=\mathbf{z}^T(n-1)\mathbf{c}+w(n).$$

The prediction error is

$$e(n)=x(n)-\mathbf{z}^T(n-1)\mathbf{c}.$$

Minimizing

$$E(\mathbf{c})=\sum_{n=N_i}^{N_f}e^2(n)$$

leads to a linear normal equation:

$$\boxed{\hat{\mathbf{R}}_z\mathbf{c}=\hat{\mathbf{r}}_z}.$$

But in most signal modeling problems, $w(n)$ is unknown. We replace it by the residual $e(n)$, which makes the error recursively depend on the unknown parameters:

$$e(n)=-\sum_{k=1}^{Q}\hat d_ke(n-k)+x(n)+\sum_{k=1}^{P}\hat a_kx(n-k).$$

Now $E(\mathbf{c})$ is no longer quadratic. Nonlinear optimization, such as a Gauss-Newton method, is needed.

## 4.7 Why ARMA Estimation Is Delicate

ARMA estimation has several practical difficulties:

| Difficulty | Explanation |
|-----------|-------------|
| Unknown excitation | $w(n)$ is not observed, so MA parameters are hidden behind residuals |
| Nonlinear objective | For $Q>0$, the residual depends recursively on $d_k$ |
| Local minima | The least-squares surface may have multiple minima |
| Stability/invertibility constraints | The estimated model should be stable and preferably minimum phase |
| Model order ambiguity | Different $P,Q$ combinations may fit the same finite data similarly |

This is why pure AR modeling is often used even when ARMA would be more compact: AR estimation is much easier and more numerically robust.

## 4.8 PZ versus High-Order AR Approximation

A high-order AR model can approximate many spectra, but it may need many poles to represent a spectral valley or notch. A PZ model can place zeros directly at valleys and therefore may match the spectrum better with fewer parameters.

> ![Figure 4.1](./CourseADSP2026/Fig/Chapter_4/fig_6_1_textbook_fig_9_13_p466.png)
>
> *Figure 4.1 (Textbook Fig. 9.13, p. 466): Comparison of a PZ(4,2) model and an AP(10) model for estimating the PSD of an ARMA(4,2) process. The pole-zero model better captures both peaks and valleys when the true signal has a pole-zero structure.*

The teaching point is not that PZ models are always better. The point is:

> A model with zeros can represent spectral valleys naturally; an all-pole model must imitate valleys indirectly by using many poles.

# §5 AR Modeling Methods

> 📖 Textbook §9.2 (Estimation of All-Pole Models); §4.2 (All-Pole Models); Chapter 3 lecture for detailed Levinson, covariance, modified covariance, and Burg recursions

## 5.1 AR Modeling Problem

The AR($P$) model is

$$\boxed{x(n)+\sum_{k=1}^{P}a_kx(n-k)=w(n)}.$$

Its transfer function is

$$H(z)=\frac{1}{A(z)}=\frac{1}{1+\sum_{k=1}^{P}a_kz^{-k}}.$$

The PSD is

$$\boxed{R_x(e^{j\omega})=\frac{\sigma_w^2}{|A(e^{j\omega})|^2}}.$$

The goal is to estimate $a_1,\ldots,a_P$ and $\sigma_w^2$.

AR modeling is equivalent to linear prediction because the residual

$$e(n)=x(n)+\sum_{k=1}^{P}a_kx(n-k)$$

should be white if the AR model is correct.

## 5.2 Yule-Walker / Autocorrelation Method

For an AR($P$) process, the autocorrelation satisfies

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=\sigma_w^2\delta(l).$$

For $l=1,2,\ldots,P$,

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=0.$$

In matrix form,

$$\boxed{\mathbf{R}_P\mathbf{a}_P=-\mathbf{r}_P},$$

where

$$\mathbf{R}_P=\begin{bmatrix}
r_x(0)&r_x(-1)&\cdots&r_x(1-P)\\
r_x(1)&r_x(0)&\cdots&r_x(2-P)\\
\vdots&\vdots&\ddots&\vdots\\
r_x(P-1)&r_x(P-2)&\cdots&r_x(0)
\end{bmatrix},$$

and

$$\mathbf{r}_P=[r_x(1),r_x(2),\ldots,r_x(P)]^T.$$

Once $\mathbf{a}_P$ is found,

$$\boxed{\sigma_w^2=r_x(0)+\sum_{k=1}^{P}a_kr_x(-k)}.$$

In practice, $r_x(l)$ is unknown and is replaced by $\hat r_x(l)$.

### Strengths

- Leads to a Hermitian Toeplitz system.
- Can be solved efficiently by Levinson-Durbin recursion.
- Produces a stable all-pole model under standard positive-definite autocorrelation assumptions.

### Weaknesses

- Uses estimated autocorrelation, which may be biased by finite-window effects.
- Zero-padding/windowing assumptions can blur spectral details.
- For short data records, peak locations and widths may be inaccurate.

## 5.3 Covariance Method

The covariance method minimizes the finite-data forward prediction error over a valid sample interval:

$$E_P=\sum_{n=P}^{N-1}\left|x(n)+\sum_{k=1}^{P}a_kx(n-k)\right|^2.$$

Unlike the autocorrelation method, it does not use zero-valued samples outside the data record.

This often gives better resolution for short records.

However, the normal equation matrix is not Toeplitz, so Levinson-Durbin cannot be used directly. Also, stability is not automatically guaranteed.

### Strengths

- Good resolution for short records.
- Avoids artificial zero extension.

### Weaknesses

- Matrix is not Toeplitz.
- More expensive than Yule-Walker.
- Estimated all-pole model may be unstable.

## 5.4 Modified Covariance Method

The modified covariance method uses both forward and backward prediction errors.

Forward error:

$$e^f(n)=x(n)+\sum_{k=1}^{P}a_kx(n-k).$$

Backward error:

$$e^b(n)=x(n-P)+\sum_{k=1}^{P}a_k^\ast x(n-P+k).$$

The combined criterion is

$$\boxed{E_P^{fb}=\sum_n\left(|e^f(n)|^2+|e^b(n)|^2\right)}.$$

Using both directions tends to reduce spectral bias. It is especially useful for narrowband signals and short data records.

### Strengths

- Often provides high-resolution spectral estimates.
- Reduces some peak-shift effects.
- Uses both time directions.

### Weaknesses

- More computationally involved.
- Still does not automatically guarantee stability unless implemented carefully.

## 5.5 Burg / Lattice Parameter Method

The Burg method estimates reflection coefficients directly by minimizing forward and backward prediction errors stage by stage.

At order $m$, it chooses a reflection coefficient $k_m$ that reduces the combined forward/backward error.

A key advantage is that if

$$|k_m|<1\qquad \text{for all }m,$$

then the resulting AR model is stable.

This is why Burg-type methods are popular in parametric spectrum estimation.

### Strengths

- Produces stable models.
- Does not require explicit autocorrelation estimation.
- Often gives high resolution for short data records.

### Weaknesses

- Can produce spectral line splitting.
- Can have frequency bias in difficult cases.
- Sensitive to model order.

## 5.6 Comparing AR Estimation Methods

| Method | Main Criterion | Matrix Structure | Stability Guarantee | Typical Use |
|--------|---------------|------------------|--------------------|-------------|
| Yule-Walker / autocorrelation | Match autocorrelation / minimize windowed prediction error | Toeplitz | Usually yes | General AR modeling; efficient computation |
| Covariance | Minimize forward prediction error on valid data | Non-Toeplitz | No | Short data, high resolution |
| Modified covariance | Minimize forward + backward errors | Non-Toeplitz | No, unless constrained | Narrowband spectra; reduced bias |
| Burg | Stagewise forward/backward error minimization | Lattice recursion | Yes if $\lvert k_m\rvert<1$ | Stable high-resolution AR modeling |

A practical teaching summary is:

- Use Yule-Walker when you want efficiency and stability.
- Use covariance or modified covariance when short-record resolution matters.
- Use Burg when stable high-resolution AR spectra are desired.

## 5.7 Model Order Selection for AR Models

The model order $P$ controls the number of poles.

If $P$ is too small:

- the spectrum is oversmoothed,
- narrow peaks may be missed,
- the residual remains colored.

If $P$ is too large:

- artificial peaks may appear,
- the model fits noise,
- the estimated spectrum may become unstable or misleading.

Common order-selection criteria include AIC and MDL, but the textbook emphasizes that such criteria are of limited value when real data are short, noisy, or not generated exactly by the assumed model.

A robust workflow is:

1. inspect ACS/PACS and periodogram,
2. try several candidate orders,
3. check residual whiteness,
4. check whether the application performance improves.

# §6 MA Modeling Methods

> 📖 Textbook §4.3 (All-Zero Models); §9.3 (Pole-Zero Estimation Ideas Applied to MA/PZ Cases)

## 6.1 MA Modeling Problem

The MA($Q$) model is

$$\boxed{x(n)=\sum_{k=0}^{Q}d_kw(n-k)}.$$

Its transfer function is

$$H(z)=D(z)=d_0+d_1z^{-1}+\cdots+d_Qz^{-Q}.$$

The PSD is

$$\boxed{R_x(e^{j\omega})=\sigma_w^2|D(e^{j\omega})|^2}.$$

Unlike AR modeling, MA modeling is not naturally a linear prediction problem in the observed samples. The excitation $w(n)$ is hidden.

This makes direct MA parameter estimation harder than AR parameter estimation.

## 6.2 Autocorrelation Cutoff Property

Because an MA($Q$) process depends only on $Q+1$ consecutive white-noise samples, its autocorrelation satisfies

$$\boxed{r_x(l)=0,\qquad |l|>Q.}$$

This property is the most important diagnostic for MA order selection.

For example, if the sample autocorrelation is significant at lags 0, 1, 2 and approximately zero after lag 2, an MA(2) model may be appropriate.

In practice, sample autocorrelations are noisy, so "cutoff" means statistically insignificant rather than exactly zero.

## 6.3 Spectral Factorization Method

If the MA autocorrelation sequence is known, we can form the finite polynomial

$$R_x(z)=\sum_{l=-Q}^{Q}r_x(l)z^{-l}.$$

For an MA process,

$$R_x(z)=\sigma_w^2D(z)D^\ast\left(\frac{1}{z^\ast}\right).$$

Therefore, estimating $D(z)$ becomes a spectral factorization problem.

The important subtlety is nonuniqueness. A given magnitude-squared spectrum can correspond to multiple zero placements. The minimum-phase factor is usually selected because it gives an invertible MA model.

### Steps

1. Estimate $r_x(l)$ for $|l|\le Q$.
2. Form $R_x(z)$.
3. Factor $R_x(z)$ into conjugate-reciprocal zero pairs.
4. Select the zeros inside the unit circle to obtain the minimum-phase $D(z)$.
5. Estimate $\sigma_w^2$ from the gain.

## 6.4 Durbin's Method for MA Modeling

Durbin's method is a practical two-stage approximation.

### Step 1: Fit a High-Order AR Model

Fit an AR model of order $P'$ to the data, where $P'$ is much larger than the desired MA order $Q$. A common rule of thumb is

$$P'\ge 4Q.$$

This high-order AR model approximates the spectrum of the data.

### Step 2: Convert the Approximation into an MA Model

The high-order AR coefficients or impulse-response approximation are then used to estimate a lower-order MA($Q$) model.

Conceptually:

1. high-order AR captures the observed spectrum,
2. its impulse-response or correlation information is treated as a target,
3. an MA($Q$) model is fitted to that target.

### Why This Works

A sufficiently high-order AR model can approximate many smooth spectra. Once an approximate spectrum is available, the MA model can be obtained indirectly.

### Limitations

- The first-stage AR order affects the result.
- If the high-order AR model is poor, the MA estimate is poor.
- MA spectral factorization still requires a minimum-phase/invertibility choice.

## 6.5 MA versus AR in Practice

| Feature | AR Model | MA Model |
|---------|----------|----------|
| Parameter estimation | Usually linear and efficient | More difficult; often indirect |
| Spectrum | Good for peaks | Good for notches |
| ACS | Tails off | Cuts off |
| PACS | Cuts off | Tails off |
| Stability | Must check poles | Always stable as FIR |
| Invertibility | Usually not an issue for all-pole model | Need zeros inside unit circle for stable inverse |

A common practical strategy is to start with AR modeling because it is simpler, then move to MA or ARMA only if the data clearly contain spectral notches or AR residuals remain structured.

# §7 Parametric Power Spectrum Estimation

> 📖 Textbook §9.4.1 (Spectral Estimation)

## 7.1 From Model Parameters to PSD

Once a model has been estimated, the PSD is obtained by substituting the estimated coefficients into the model spectrum.

For an ARMA model,

$$\boxed{\hat R_x(e^{j\omega})=\hat\sigma_w^2\frac{|\hat D(e^{j\omega})|^2}{|\hat A(e^{j\omega})|^2}}.$$

For an AR model,

$$\boxed{\hat R_x(e^{j\omega})=\frac{\hat\sigma_w^2}{|\hat A(e^{j\omega})|^2}}.$$

This is a parametric spectrum estimate.

## 7.2 Why Parametric Spectra Can Have High Resolution

Classical nonparametric estimators such as the periodogram, Bartlett, Welch, and Blackman-Tukey methods estimate the spectrum directly from finite data. Their resolution is strongly tied to record length and windowing.

Parametric methods impose a model. If the model is appropriate, the estimated poles and zeros extrapolate the autocorrelation beyond the observed lags.

This can yield high spectral resolution from short records.

The key phrase is **if the model is appropriate**.

Parametric methods can perform very well for signals that are actually close to AR, MA, or ARMA. They can perform poorly when the true signal does not follow the assumed model.

## 7.3 Model Mismatch

Model mismatch occurs when the assumed model family cannot represent the true signal well.

Examples:

| True Signal Structure | Bad Model Choice | Possible Consequence |
|----------------------|------------------|----------------------|
| Contains spectral notch | Pure AR model | Notch poorly represented |
| Contains sharp resonant peaks | Low-order MA model | Peaks oversmoothed |
| Nonstationary trend | Stationary ARMA model | Spurious low-frequency structure |
| Long-memory process | Short-memory ARMA model | Slow correlations not captured |
| Mixed line and continuous spectrum | Pure continuous PSD model | Harmonic lines smeared |

Figure 4.1 earlier showed a PZ model outperforming a higher-order AP model when the target process has ARMA structure. This is a model-mismatch lesson, not just a numerical example.

## 7.4 Comparing All-Pole PSD Estimation Methods

The textbook compares several all-pole PSD estimation methods using Monte Carlo simulations.

> ![Figure 7.1](./CourseADSP2026/Fig/Chapter_4/fig_8_1_textbook_fig_9_14_p468.png)
>
> *Figure 7.1 (Textbook Fig. 9.14, p. 468): Monte Carlo comparison of all-pole PSD estimation techniques using short data records. Yule-Walker, LS, forward/backward LS, Burg, and Itakura-Saito approaches have different bias/variance behavior.*

The main teaching message is:

- windowing-based approaches can lose resolution,
- short records produce variability across realizations,
- all-pole methods can reveal narrow peaks, but they can also create spurious structure,
- the method and model order should be validated with residual diagnostics.

## 7.5 Linear Prediction Prewhitening for Spectrum Estimation

A very useful hybrid approach is linear prediction prewhitening.

The idea is:

1. Fit an AP($P$) model to the data.
2. Filter the data by the prediction error filter $A(z)$ to obtain residuals.
3. Estimate the residual PSD using a nonparametric method.
4. Postcolor the residual PSD by the inverse of the whitening operation.

Mathematically, if

$$e(n)=A(z)x(n),$$

then

$$R_e(e^{j\omega})=|A(e^{j\omega})|^2R_x(e^{j\omega}).$$

Therefore,

$$\boxed{\hat R_x(e^{j\omega})=\frac{\hat R_e(e^{j\omega})}{|A(e^{j\omega})|^2}}.$$

> ![Figure 7.2](./CourseADSP2026/Fig/Chapter_4/fig_8_2_textbook_fig_9_15_p469.png)
>
> *Figure 7.2 (Textbook Fig. 9.15, p. 469): Nonparametric PSD estimation using linear prediction prewhitening. The AR model reduces spectral dynamic range; the residual spectrum is estimated nonparametrically; postcoloring restores the original scale.*

This approach is often practical because it combines strengths:

| Component | Role |
|----------|------|
| AR prewhitening | Reduces strong spectral coloring and leakage |
| Nonparametric residual PSD | Captures remaining structure without forcing a pure AR model |
| Postcoloring | Restores the original spectral envelope |

## 7.6 Main Defects of Parametric Spectrum Estimation

The outline emphasizes two major defects.

### Defect 1: Correct Model Assumption Is Not Guaranteed

A parametric spectrum estimate is only as reliable as the assumed model family. If the real signal is not close to AR, MA, or ARMA, the fitted parameters may produce a misleading spectrum.

This is especially dangerous because parametric spectra often look smooth and convincing.

### Defect 2: Model Order Is Difficult

The order controls the number of poles and zeros. It is not just a minor parameter.

Too low an order causes underfitting:

- peaks disappear,
- notches are missed,
- residuals remain colored.

Too high an order causes overfitting:

- spurious peaks appear,
- noise is modeled as signal,
- estimates vary strongly across data records.

Therefore, a careful modeler must check both the fitted spectrum and the residual.

## 7.7 Practical Diagnostic Checklist

When using parametric spectrum estimation, use the following checklist.

| Diagnostic | Question |
|-----------|----------|
| ACS/PACS | Do the empirical ACS/PACS match the model family signature? |
| Residual whiteness | Is $e(n)$ approximately white? |
| Stability | Are estimated poles inside the unit circle? |
| Invertibility | Are estimated zeros inside the unit circle if a stable inverse is needed? |
| Order sensitivity | Does the result change drastically when $P,Q$ change slightly? |
| Physical plausibility | Are the peaks/notches meaningful for the application? |
| Comparison baseline | Does the estimate agree reasonably with a nonparametric spectrum? |

A parametric estimate should not be accepted solely because it has higher apparent resolution.

## 8.3 Suggested Teaching Flow

1. Start with the white-noise-through-filter picture: $x(n)=H(z)w(n)$.
2. Use Figure 1.2 to explain why modeling, whitening, and prediction are the same idea viewed from different directions.
3. Introduce $H(z)=D(z)/A(z)$ and immediately classify AP, AZ, and PZ models.
4. Emphasize the physical meaning of poles and zeros: poles create peaks, zeros create notches.
5. Use AP(2) to connect pole geometry, impulse response, ACS, and spectrum.
6. Teach the ACS/PACS signatures before teaching algorithms. This prevents students from treating model choice as arbitrary.
7. Present AR estimation first because it is linear and connects directly to Chapter 3.
8. Present ARMA estimation as harder because the excitation is hidden and the least-squares surface is nonlinear.
9. End with parametric PSD estimation, stressing both its power and its danger: high resolution is useful only when the model is credible.

## 8.4 Minimal Board Derivation Plan

A concise blackboard derivation can follow this sequence.

### Step 1: White Noise Through a Filter

$$x(n)=h(n)\ast w(n),\qquad R_x(e^{j\omega})=\sigma_w^2|H(e^{j\omega})|^2.$$

### Step 2: Rational Model

$$H(z)=\frac{D(z)}{A(z)}=\frac{d_0+d_1z^{-1}+\cdots+d_Qz^{-Q}}{1+a_1z^{-1}+\cdots+a_Pz^{-P}}.$$

### Step 3: AR Special Case

$$x(n)+\sum_{k=1}^{P}a_kx(n-k)=w(n).$$

Multiply by $x^\ast(n-l)$ and take expectations:

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=\sigma_w^2\delta(l).$$

For $l=1,\ldots,P$:

$$\mathbf{R}_P\mathbf{a}_P=-\mathbf{r}_P.$$

### Step 4: ARMA Long-Lag Equation

For ARMA, define $c(n)=A(z)x(n)=D(z)w(n)$. Since $c(n)$ is MA($Q$), for $l>Q$:

$$r_x(l)+\sum_{k=1}^{P}a_kr_x(l-k)=0.$$

This gives the MYWE method for the AR part.

### Step 5: PSD Estimate

$$\hat R_x(e^{j\omega})=\hat\sigma_w^2\frac{|\hat D(e^{j\omega})|^2}{|\hat A(e^{j\omega})|^2}.$$

End the lecture by asking students: *Does the residual look white?* If the answer is no, the model is not finished.
